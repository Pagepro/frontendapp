26.5 Ticket description

Trzeba przygotować ekran szczegółów ticketu.
To ekran dla klienta czyli ma nieco okrojone funkcjonalności.

Wygląda to tak:
- published by: dane, data dodania, data aktualizacji,
- ticket status: np. Done - to jest select
- assigned to: to jest select
- opis ticketu
- info do jakiego template przypisany jest ticket
- sekcja komentarzy

Każdy komentarz to:
- treść
- podpis imię i nazwisko
- podpis: data i czas dodania

Pod sekcją komentarzy - formularz dodawnia komentarza.
